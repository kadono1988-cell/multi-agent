## 1. システム概要

建設プロジェクトの意思決定を、**4人のAI専門家が議論する5ラウンド形式**でサポートするプラットフォームです。

### 4人の専門家

| 役職 | 担当分野 |
|------|---------|
| **PM（現場プロジェクトマネージャー）** | 現場の実行可能性・工程・安全 |
| **CFO（財務責任者）** | コスト・キャッシュフロー・財務リスク |
| **COO（運営責任者）** | 契約・ステークホルダー・ブランド |
| **CEO（最高経営責任者）** | 全体統合・最終判断・不足情報の特定 |

### 議論の流れ（5ラウンド）

```
Round 1  PM/CFO/COO が初期分析・リスク提示
   ↓
Round 2  専門家同士が互いの意見に反論・深化
   ↓
Round 3  CEO が不足情報を最大3つ特定して質問
   ↓
   ↑ ユーザーが追加情報を入力（右パネル）
Round 4  追加情報を踏まえた専門家の最終見解
   ↓
Round 5  CEO が最終決定・実行指示を発出
```

## 2. 起動方法

### デモモード（APIキー不要）

```bash
npm install   # 初回のみ
npm run dev
```

ブラウザで `http://localhost:5173` を開いてください。

### 本番モード（実際のAIを使う場合）

1. `.env` ファイルに以下を設定する（README.md 参照）

```
VITE_GEMINI_API_KEY=あなたのGemini APIキー
VITE_SUPABASE_URL=SupabaseプロジェクトURL
VITE_SUPABASE_ANON_KEY=SupabaseのAnonキー
```

2. Supabase に `supabase/schema.sql` を実行してテーブルを作成する
3. `npm run dev` で起動

## 4. プロジェクト管理

### プロジェクトの作成

1. 左サイドバー上部の「＋」ボタンをクリック
2. 以下のフィールドを入力（**プロジェクト名のみ必須**）

| フィールド | 説明 | 例 |
|-----------|------|-----|
| プロジェクト名 | 必須 | 渋谷駅前オフィスビル新築工事 |
| 施主 | 発注者・クライアント名 | 株式会社○○デベロップメント |
| 重要度 | high / medium / low | high |
| 予算 | 概算金額 | 12億円 |
| 工期 | 開始〜終了 | 2024年4月〜2025年3月 |
| 規模 | 延床面積など | 延床5,200m² |
| 建物用途 | 用途区分 | 商業・オフィス複合 |
| 場所 | 所在地 | 東京都渋谷区 |
| プロジェクト概要 | 現状・課題・背景（AI議論の出発点になる） | 工期が3ヶ月遅延しており… |
| 備考 | 特記事項 | 隣接建物との離隔制限あり |

3. 「保存」ボタンで登録

> **Template CSV ボタン**: フォーム画面から CSV テンプレートをダウンロードできます（Excel で編集後に手入力する際の参考用）

### プロジェクトの編集

1. プロジェクト一覧からクリックしてプロジェクトを選択
2. メインエリアに表示されるプロジェクト詳細の「編集」ボタンをクリック

### プロジェクトの削除

1. 対象プロジェクトを選択して編集画面を開く
2. フォーム下部の「削除」ボタン（赤字）をクリック
3. 確認ダイアログで「OK」

> ⚠️ 削除は取り消せません。関連するセッション履歴も失われます。

### プロジェクトの検索・フィルター

- **キーワード検索**: サイドバーの検索ボックスに名前や施主名を入力すると即時フィルタリング
- **重要度フィルター**: 「全て / 高 / 中 / 低」タブで絞り込み

## 5. 議論セッションの進め方

### ステップ 1: プロジェクトを選択

サイドバーのプロジェクト一覧をクリックします。メインエリアにプロジェクト詳細とテーマ選択グリッドが表示されます。

### ステップ 2: 議論テーマを選択

| テーマ | 用途 |
|--------|------|
| ⏰ 遅延回復戦略 | 工程遅延が発生・懸念される場合 |
| 🚦 Go / No-Go 判断 | プロジェクト継続の可否を判断したい場合 |
| 📐 設計変更への対応 | 設計変更の影響・リスクを評価したい場合 |
| 💬 個別具体の相談 | 上記に当てはまらない任意の課題 |
| カスタムテーマ | 自分で追加・保存したテーマ（後述） |

### ステップ 3: セットアップ入力（任意）

テーマを選ぶとセットアップ画面に移ります。空欄でも議論は開始できますが、情報を入れるほど議論の精度が上がります。

| 項目 | 内容 |
|------|------|
| 相談内容 | 具体的な状況・事象の説明 |
| 制約事項 | 変更できない条件・制約 |
| 目指すゴール | 今回の議論で達成したいこと |

### ステップ 4: 議論を進める

「🚀 議論を開始する」をクリックすると Round 1 が始まります。

- **AI応答はリアルタイムで表示**: テキストがタイプライター形式で流れます
- **次のラウンドへ**: メインエリア右上の「Round N へ進む」ボタン
- **Round 3 では追加情報を入力**: 右パネルの「CEO要請への回答」欄に情報を入力して「回答を送信」→ Round 4 へ進む

### ステップ 5: 最終判断を確認・ダウンロード

Round 5 完了後、CEOの最終判断が表示されます。

- **「議論メモをDL」ボタン**: 全5ラウンドの議論内容を `.txt` ファイルでエクスポート
- ファイル名: `decision_memo_プロジェクト名_日付.txt`

## 6. 専門家チームの設定

左サイドバーの「専門家チーム設定」タブで管理します。

### 既存エージェントの編集

1. エージェントカードの「編集」ボタンをクリック
2. 以下を変更可能

| 項目 | 説明 |
|------|------|
| 表示名 (Role ID) | PM, CFO など。議論内の発言者名に使われる |
| フルネーム | カード上の表示名 |
| 役割・関心事 | AI への System Prompt。このエージェントが重視する視点を記述 |
| 話し方・スタイル | 発言の雰囲気・スタイルを指定 |

3. 「保存」ボタンで確定（ローカルと Supabase に自動同期）

### エージェントの追加

「＋ 新規エージェント追加」カードをクリック → フォームを入力 → 保存

> デフォルト4名（PM/CFO/COO/CEO）に加えて、例えば「法務担当顧問（Legal）」「安全担当（Safety）」など自由に追加できます。

### エージェントの有効・無効切り替え

カード下部のトグルスイッチで「参加中 / 不参加」を切り替えられます。不参加にしたエージェントは議論に加わりません（CEO は常に参加）。

### エージェントの削除

カード右上の赤いゴミ箱アイコンをクリック → 確認ダイアログで「OK」

> デフォルトの4エージェントも削除できますが、削除するとデフォルト設定に戻る場合があります。

## 7. ナレッジベース（過去事例）管理

左サイドバーの「知見・過去事例」タブで管理します。

過去事例は議論中に**自動でRAG（検索拡張生成）として引用**され、より精度の高い分析を実現します。

### 事例の閲覧

登録された事例がカード形式で一覧表示されます。各カードには以下が表示されます。
- タイトル / プロジェクト種別タグ
- 概要（状況・対応策）
- 最終結果・教訓

### 事例の追加（本番モードのみ）

1. 「＋ 事例を追加」ボタンをクリック
2. 以下を入力

| 項目 | 必須 | 説明 |
|------|------|------|
| タイトル | ✅ | 事例の名称 |
| プロジェクト種別 | — | 商業施設、公共工事 など |
| 概要 | — | 状況・課題・対応策 |
| 最終結果・教訓 | — | 結果と今後への示唆 |

3. 「保存」で Supabase に登録

### 事例の編集・削除

各カード右上の「編集」「ゴミ箱」ボタンから操作します。

> デモモードでは事例の編集・追加は行えません。

## 8. 議論テーマのカスタマイズ

デフォルト4テーマに加えて、独自テーマを追加・保存できます。カスタムテーマは**ブラウザの localStorage に永続保存**されるため、次回起動後も引き続き使えます。

### デフォルト4テーマ

| テーマ | 向いている場面 |
|--------|-------------|
| ⏰ 遅延回復戦略 | 工程が遅延している・遅延しそうな場合 |
| 🚦 Go / No-Go 判断 | プロジェクト継続の可否を判断したい場合 |
| 📐 設計変更への対応 | 設計変更の影響・コスト・リスクを評価したい場合 |
| 💬 個別具体の相談 | 上記3つに当てはまらない任意の課題 |

### カスタムテーマの追加手順

1. プロジェクト詳細画面のテーマ選択グリッド、右上の「＋ テーマを追加」をクリック
2. **テーマ名**（必須）と**説明**を入力して「保存」
3. テーマグリッドに追加され、以降ずっと使えます

> 例: `🏗️ 協力会社交渉戦略`（説明: 協力会社との単価・工程交渉の進め方を議論）

### カスタムテーマの削除

テーマカード右上の小さなゴミ箱アイコンをクリック → 即時削除（デフォルト4テーマは削除不可）。

## 9. 便利な機能

### セッション履歴の閲覧（本番モード）

プロジェクト詳細画面の下部に「過去の議論履歴」セクションが表示されます。

- 過去8件の議論が日付付きで一覧表示
- クリックすると全ラウンドの内容を閲覧可能（読み取り専用）
- 「議論メモをDL」で過去セッションのメモもエクスポートできます

### ページリロード耐性

進行中のセッションはブラウザの `sessionStorage` に自動保存されます。

- 誤ってブラウザを更新した場合、同じタブを開き直すと**自動的に復元**されます
- タブを閉じた場合はセッションが失われます

### 議論メモのダウンロード

- Round 5 完了後、メインエリア右上に「📥 議論メモをDL」ボタンが表示されます
- ダウンロードされる `.txt` ファイルの構成:
  - プロジェクト情報ヘッダー
  - Round 1〜5 の全発言（エージェント名付き）

### モバイル・タブレット対応

| 画面幅 | 表示 |
|--------|------|
| ≥ 1100px | 通常の3カラム表示 |
| ≤ 1100px | 右パネルを非表示（2カラム） |
| ≤ 640px | ボトムナビゲーション形式 |

## 10. デモモードと本番モードの違い

| 機能 | デモモード | 本番モード |
|------|-----------|-----------|
| プロジェクト | サンプル2件（固定） | 自由に作成・編集・削除 |
| AI議論 | 事前用意したサンプル応答を表示 | Gemini AI がリアルタイム生成（ストリーミング） |
| ナレッジベース | 閲覧のみ | 追加・編集・削除 |
| 過去セッション履歴 | 非表示 | 表示・閲覧 |
| データ保存 | なし（リロードでリセット） | Supabase に永続保存 |
| APIキー | 不要 | Gemini API Key + Supabase 必要 |

デモモードはAPIキー・インターネット接続なしで動作確認するためのものです。実運用には本番設定が必要です。

## 11. よくある質問・トラブル

### Q. 「議論の開始に失敗しました」と表示される

**A.** 主な原因は以下の通りです。

1. **Gemini API キーが間違っている** — `.env` ファイルの `VITE_GEMINI_API_KEY` を確認
2. **APIの無料枠を超えた** — [Google AI Studio](https://aistudio.google.com/) でクォータを確認
3. **Supabase の接続情報が誤っている** — `.env` の URL と Anon Key を確認

### Q. エージェントの設定が保存されない

**A.** Supabase の `expert_agents` テーブルが未作成の場合、ブラウザの `localStorage` に保存されます（動作はします）。永続化したい場合は `supabase/schema.sql` を実行してください。

### Q. ブラウザをリロードしたら議論が消えた

**A.** セッションは `sessionStorage` に保存されるため、**同じタブ**で更新した場合は復元されます。タブを閉じた場合は復元できません。

### Q. ページが白画面になって動かない

**A.** ブラウザの開発者ツール（F12）のコンソールでエラーを確認してください。日本語パスに起因する Vite の問題の場合は、プロジェクトを ASCII パスのフォルダ（例: `C:/projects/multi-agent/`）に移動すると解決します。

### Q. デモモードで5ラウンド全部試せますか？

**A.** はい。デモモードでも全5ラウンド分のサンプル議論が用意されています。「銀座中央通り再開発」プロジェクトを選んで試してください。

### Q. 右パネルが表示されない

**A.** ブラウザ幅が 1100px 以下になると右パネルは自動的に非表示になります（CEO への回答入力は、ブラウザを広げると右パネルに表示されます）。
